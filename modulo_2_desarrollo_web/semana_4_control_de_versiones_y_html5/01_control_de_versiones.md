# Sistemas de Control de Versiones

## ¿Qué son los sistemas de control de versiones?

Los **sistemas de control de versiones (SCV)** son herramientas de software que ayudan a los equipos a gestionar los cambios en el código de un proyecto a lo largo del tiempo.

Permiten:
- Realizar un seguimiento de todas las modificaciones del código.
- Saber **qué** se cambió, **cuándo** se cambió y **quién** lo cambió.
- Volver a una versión anterior del código en caso de errores o fallos.

## Algunos sistemas de control de versiones

- **Git**
- **Apache Subversion (SVN)**
- **Mercurial**
- **CVS**

## ¿Qué es Git?

**Git** es el sistema de control de versiones más utilizado en el mundo.

Características principales:
- Es un proyecto de **código abierto**, desarrollado en 2005 por **Linus Torvalds**, creador del kernel de Linux.
- Es un sistema de control de versiones **distribuido**, lo que significa que cada desarrollador tiene una copia completa del historial del proyecto.
- Permite trabajar con **ramas**, facilitando el desarrollo de nuevas funcionalidades sin afectar el código principal.
- Está diseñado para ofrecer **alto rendimiento**, **seguridad** y **flexibilidad**.

## ¿Qué es GitHub?

**GitHub** es una plataforma web creada para que los desarrolladores puedan subir el código de sus aplicaciones y permitir que otros usuarios:

- Vean el código
- Lo descarguen
- Colaboren agregando cambios

Características:
- Utiliza el sistema de control de versiones **Git**.
- Permite almacenar todas las versiones del código.
- Fue **comprada por Microsoft en 2018**.
- Es **gratuita para proyectos de código abierto**, donde cualquier persona puede ver y descargar el código.
- Es **de pago para proyectos privados**, donde solo usuarios con permisos pueden acceder al código.
